title: Quickstart
description: Add your first RareUI component in under 5 minutes. Choose between the CLI (recommended) or manual copy-paste.
---

This guide walks you through adding a RareUI component to an existing Next.js project. If you don't have a project yet, follow [Install Next.js](/installation/install-nextjs) first.

## Option 1: CLI (recommended)

The CLI handles file placement, dependency installation, and configuration automatically.

<Steps>
  <Step title="Initialize RareUI">
    Run the `init` command from your project root. This creates a `components.json` file that configures paths and aliases.

    <CodeGroup>

    ```bash npm
    npx rareui@latest init
    ```

    ```bash pnpm
    pnpm dlx rareui@latest init
    ```

    ```bash yarn
    yarn dlx rareui@latest init
    ```

    ```bash bun
    bunx rareui@latest init
    ```

    </CodeGroup>

    You'll see a confirmation prompt if `components.json` already exists. After initialization, the file contains your project's path aliases and registry configuration.
  </Step>

  <Step title="Add a component">
    Add the `liquid-button` component — one of the simplest to get started with.

    <CodeGroup>

    ```bash npm
    npx rareui@latest add liquid-button
    ```

    ```bash pnpm
    pnpm dlx rareui@latest add liquid-button
    ```

    ```bash yarn
    yarn dlx rareui@latest add liquid-button
    ```

    ```bash bun
    bunx rareui@latest add liquid-button
    ```

    </CodeGroup>

    The CLI downloads the component source to `components/rareui/` and installs any required dependencies (`framer-motion`, `clsx`, `tailwind-merge`) automatically.
  </Step>

  <Step title="Use the component">
    Import and render the component in any page or layout file.

    ```tsx title="app/page.tsx"
    import { LiquidButton } from "@/components/rareui/liquid-button"

    export default function Page() {
      return (
        <main className="flex min-h-screen items-center justify-center">
          <LiquidButton>Get started</LiquidButton>
        </main>
      )
    }
    ```
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    Open `http://localhost:3000` to see the liquid button rendered with its gooey animation.
  </Step>
</Steps>

## Option 2: Manual copy-paste

If you prefer not to use the CLI, you can copy component code directly from the documentation.

<Steps>
  <Step title="Install dependencies">
    Install the packages that RareUI components depend on.

    <CodeGroup>

    ```bash npm
    npm install framer-motion clsx tailwind-merge
    ```

    ```bash pnpm
    pnpm add framer-motion clsx tailwind-merge
    ```

    ```bash yarn
    yarn add framer-motion clsx tailwind-merge
    ```

    ```bash bun
    bun add framer-motion clsx tailwind-merge
    ```

    </CodeGroup>
  </Step>

  <Step title="Create the utility function">
    Create `lib/utils.ts` with the `cn()` helper. All RareUI components use this for conditional class merging.

    ```ts title="lib/utils.ts"
    import { type ClassValue, clsx } from "clsx"
    import { twMerge } from "tailwind-merge"

    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs))
    }
    ```
  </Step>

  <Step title="Create the component file">
    Create a file at `components/rareui/liquid-button.tsx` and paste the component source from the [Liquid Button component page](/components/liquid-button).

    The component will use your `@/lib/utils` import path, so make sure your `tsconfig.json` has the `@/*` path alias configured:

    ```json title="tsconfig.json"
    {
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "@/*": ["./*"]
        }
      }
    }
    ```
  </Step>

  <Step title="Import and use">
    ```tsx title="app/page.tsx"
    import { LiquidButton } from "@/components/rareui/liquid-button"

    export default function Page() {
      return (
        <main className="flex min-h-screen items-center justify-center">
          <LiquidButton>Get started</LiquidButton>
        </main>
      )
    }
    ```
  </Step>
</Steps>
